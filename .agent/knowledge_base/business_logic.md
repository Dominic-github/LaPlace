## Core Business Rules

### 1. Accommodation Management
- Chủ trọ đăng tin phòng với: tiêu đề, mô tả, địa chỉ, giá, diện tích, tiện nghi, ảnh
- Trạng thái phòng: `available` → `occupied` → `maintenance`
- Phòng có tiện nghi (WiFi, Điều hòa, Nóng lạnh...) qua quan hệ many-to-many
- Ảnh phòng lưu riêng trong bảng `accommodation_images`

### 2. Booking Flow
```
Khách chọn phòng → Kiểm tra availability → Tạo booking (pending)
  → Admin xác nhận (confirmed)
    → Tạo hợp đồng (active) → Phòng chuyển sang occupied
    → Hoặc hủy (cancelled)
```
- User phải đăng nhập mới đặt được phòng
- Kiểm tra không có booking trùng ngày
- Tự động tính `totalPrice`
- Gửi email xác nhận booking

### 3. Contract Flow
```
Booking (confirmed) → Tạo Contract (active)
  → Booking chuyển completed
  → Accommodation chuyển occupied
  → Payment monthly theo contract
  → Khi hết hạn: Contract → expired, Accommodation → available
  → Hoặc chấm dứt sớm: Contract → terminated
```

### 4. Payment Flow
- Thanh toán theo booking hoặc contract
- Phương thức: Cash, Bank Transfer, Online
- Trạng thái: `pending` → `completed` / `failed`
- Lưu transactionId cho thanh toán online

### 5. Review System
- User phải có booking `completed` mới được review
- Một user chỉ review 1 lần cho 1 accommodation
- Rating 1-5 sao + bình luận
- Admin có quyền xóa review không phù hợp

### 6. User Roles (RBAC)
| Role | Quyền |
|------|-------|
| Admin | Full access - CRUD tất cả |
| Manager | Quản lý phòng trọ, booking |
| Customer | Đặt phòng, review, yêu thích |
| Landlord | Đăng tin, quản lý tin, xem thống kê |

### 7. Favorites & Messages
- User có thể yêu thích phòng (toggle)
- Nhắn tin real-time giữa user và chủ trọ qua Socket.io
